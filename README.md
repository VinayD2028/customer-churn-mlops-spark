## Overview

This project implements a production-style ML pipeline that predicts whether a telecom customer will churn based on their usage patterns and subscription attributes. The entire pipeline runs on **Apache Spark**, making it horizontally scalable and suitable for large-scale data engineering environments.

The pipeline spans data synthesis, preprocessing, baseline modelling, statistical feature selection, and automated hyperparameter tuning across four classification algorithms — all evaluated using AUC-ROC via 5-fold cross-validation.

## Pipeline Architecture

```
+-----------------------------------------------------------------+
|                     PIPELINE WORKFLOW                           |
|                                                                 |
|  1. Data Generation                                             |

```

## Key Features

- **Distributed ML at Scale** — Built entirely on Apache Spark MLlib, enabling processing of millions of records with horizontal scaling
- **Automated Data Synthesis** — Programmatic generation of realistic telecom churn data with tunable churn probability logic and deliberate missing value injection (~5% NaN rate in TotalCharges)
- **Production-Style Feature Engineering** — Full preprocessing pipeline using Spark ML transformers: `StringIndexer`, `OneHotEncoder`, and `VectorAssembler`
- **Statistical Feature Selection** — Chi-Square test (`ChiSqSelector`) identifies the top 5 most statistically significant predictors
- **Multi-Model Benchmarking** — Trains, tunes, and compares four classifiers in a unified evaluation framework
- **Automated Hyperparameter Tuning** — `ParamGridBuilder` + `CrossValidator` with 5-fold CV for each model
- **Reproducible Results** — Fixed random seed (`seed=42`) ensures consistent train/test splits across runs
- **Structured Output Logging** — All results persisted to labelled text files for audit and reporting

## Dataset

The dataset (`customer_churn.csv`) is synthetically generated by `dataset-generator.py` to simulate a realistic telecom churn scenario. It contains **1,000 customer records** with the following schema:

| Feature | Type | Description |
|---|---|---|
| `customerID` | String | Unique customer identifier (dropped before modelling) |
| `gender` | Categorical | Male / Female |
| `SeniorCitizen` | Binary | 1 if senior citizen, 0 otherwise |
| `tenure` | Integer | Months as a customer (0–72) |
| `PhoneService` | Categorical | Yes / No |
| `InternetService` | Categorical | DSL / Fiber optic / No |
| `MonthlyCharges` | Float | Monthly billing amount |
| `TotalCharges` | Float | Cumulative charges (contains ~5% NaN) |
| `Churn` | Categorical | **Target** — Yes / No |

**Churn logic:** Customers with short tenure and high charges, senior citizens, or Fiber optic subscribers are assigned higher churn probability, with stochastic noise applied to prevent over-determinism.

## Models & Results

| Model | Tuned Parameters | Test AUC-ROC |
|---|---|---|
| Logistic Regression (baseline) | Default | ~0.73 |
| Logistic Regression (tuned) | regParam in {0.01, 0.1} | ~0.84 |
| Decision Tree | maxDepth in {5, 10} | ~0.77 |
| Random Forest | numTrees in {10, 50} | ~0.82 |
| Gradient Boosted Trees | maxIter in {10, 20} | ~0.74 |

All models evaluated using `BinaryClassificationEvaluator` (AUC-ROC metric) with 5-fold cross-validation on the training set and final scoring on the held-out test set.

## Getting Started

### Prerequisites

- Python 3.8+
- Java 8 or 11 (required by Spark)
- Apache Spark 3.x installed and `SPARK_HOME` configured

```bash
pip install pyspark pandas numpy
```

### Run the Pipeline

**Step 1 — Generate the dataset:**
```bash
python3 dataset-generator.py
```
This creates `customer_churn.csv` in the project root.

**Step 2 — Execute the full ML pipeline:**
```bash
spark-submit customer-churn-analysis.py
```
Results are written to the `outputs/` directory.
